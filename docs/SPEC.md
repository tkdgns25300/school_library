# School Library — 기획서

## 개요

- **사용처**: 더힘스쿨 수지점 (기독교 대안학교, 1~6학년)
- **목표**: 기존 "르네상스" 도서 대여 시스템 대체
- **사용자**: 교사 다수가 단일 관리자 계정으로 공용 로그인
- **장비**: USB 바코드 스캐너

## 학교 구조

학년: 1~6.

반(`class_section`)은 총 3개:

| 반 | 소속 학년 |
|---|---|
| `junior 1` | 1~3학년 일부 |
| `junior 2` | 1~3학년 일부 |
| `senior 1` | 4~6학년 전부 |

학생은 항상 **학년 + 반** 두 정보를 함께 보유. 교사는 담당 반을 보유.

## 책 도메인

책은 두 종류:

- **한국어 도서** (`language = 'ko'`): 표시는 **단계**. 컬러는 **남색**.
- **영어 도서** (`language = 'en'`): 표시는 **레벨**. 컬러는 **초록**.

DB는 `level` text 컬럼 한 개로 통합 저장. UI에서 언어에 따라 명칭("단계"/"레벨")만 분기. 값 형식은 학교가 자유롭게 정의(예: `1`, `2단계`, `Level 3`, `AR 2.5`).

각 책에는 **표지 이미지**(`cover_image_url`)가 있음.

## 바코드

- 르네상스 바코드 사용 불가 → 시스템이 자체 발급 (Code128, `BK00001`~).
- 책 등록 시 자동 발급 + 미리보기.
- 라벨 PDF는 책 목록에서 다중 선택해 일회성 출력. 인쇄 여부는 시스템에서 추적 안 함.
- USB 스캐너는 키보드 에뮬레이션이라 `<input>` 포커스만 두면 동작.

## 페이지 (5개로 고정)

추가 페이지 없음. 사이드바 네비에 항상 이 5개만.

### 1. 운영 (`/`)

진입 시 **반 카드 3개**(`junior 1`, `junior 2`, `senior 1`)만 노출. 다른 정보(KPI, 연체 패널, 통계 등) 일체 없음.

반 클릭 → **반별 운영 화면**:

- 좌·우 1:1 두 칼럼: **한국어 도서** | **English Books** (KO/EN 배지)
- 각 칼럼 내부:
  1. 헤더 — 해당 칼럼의 대여 중 권수, 연체 권수
  2. 대여/반납 모드 토글
  3. 학생 선택(이름 검색) + 반납 예정일(대여 모드만)
  4. 담당 교사 선택
  5. 바코드 입력 필드 (자동 포커스, Enter 종료, 연속 스캔)
  6. **스캔된 책 표지 즉시 미리보기**
  7. 해당 칼럼 언어의 **현재 대여 중 도서 리스트**

**잘못된 칸 안내**: 한국어 칼럼에서 영어 책 스캔(또는 반대) → "다른 칸에서 처리하세요" 토스트, 처리 안 함.

**대여 중 리스트 정렬**: 연체 먼저 → 학년 오름차순 → 이름.

**가독성**: 학생 이름·학년이 한눈에 띄도록 강조 (이름 굵게, 학년 컬러 배지). 대여 도서가 많아도 "누가 무엇을 빌렸는지" 시선이 즉시 닿게.

### 2. 전체 통계 (`/stats`)

**반별 뷰** + **전체 뷰** (탭/세그먼트 전환).

각 모드에서:
- 한국어/영어 구분 노출
- 학생 정렬: 연체 먼저 → 학년 오름차순 → 이름
- 학생별: 현재 대여 한국어 N권/영어 N권, 연체 한국어 N권/영어 N권

### 3. 학생 명단 (`/students`)

전체 학생 목록만 (통계·반별 집계 없음 — 전체 통계 페이지 책임).

- 디폴트 정렬: 학년 오름차순 → 반(`junior 1` → `junior 2` → `senior 1`) → 이름
- 검색: 이름 / 필터: 학년, 반
- **추가 폼 입력 필드**: 이름, 학년, 반 (그 외 없음)
- 수정/삭제 (활성 대여 있는 학생은 삭제 차단)
- CSV 일괄 업로드 + UI 개별 CRUD

### 4. 교사 명단 (`/teachers`)

- 표시: 이름, 담당 반 (등록일·처리 건수 표시 X)
- 추가 폼: 이름, 담당 반
- 수정/삭제 (활성 대여 처리 이력 있으면 삭제 차단)
- CSV 일괄 업로드 + UI 개별 CRUD

### 5. 책 목록 (`/books`)

- 한국어/영어 탭으로 분리
- 각 책 카드/행에 표지 썸네일, 클릭 시 큰 미리보기
- 표시: 표지, 제목, 저자, 출판사, 학년, 단계(한국어)/레벨(영어)
- 필터: 학년, 언어, 단계/레벨
- 검색: 제목, 저자, 단계/레벨 (텍스트 매칭)
- 추가 폼: 제목/저자/출판사/학년/언어/단계·레벨/표지 URL + 자동 ID + 바코드 미리보기
- 수정/삭제 (활성 대여 있으면 차단)
- CSV 일괄 업로드 + UI 개별 CRUD
- **다중 선택 → A4 라벨 PDF 출력** (라벨 미인쇄 필터·뱃지 없음)

## 정렬 규칙 (전역 일관)

> 코드·UI·SQL 모두 이 표를 기준. SCHEMA.md에 SQL 패턴 코드 있음.

| 위치 | 정렬 |
|---|---|
| 학생 명단 | 학년 ↑ → 반(`junior 1` → `junior 2` → `senior 1`) → 이름 |
| 운영 화면 / 대여 중 리스트 | 연체 먼저 → 학년 ↑ → 이름 |
| 전체 통계 (반별·전체 뷰) | 연체 먼저 → 학년 ↑ → 이름 |

## 비기능 요구

- **반응 속도**: 스캔 후 표지·결과 표시까지 200ms 이내 체감. 연속 스캔이 끊기지 않을 것.
- **데이터 무결성**: 한 책의 활성 대여는 1개만 (DB 제약). 학생/책/교사 삭제는 활성 대여 있으면 차단.
- **가독성**: 이름·학년이 시각적으로 즉시 식별 가능해야 함.

## 우선순위

- **Phase 1 (MVP)**: 인증, 5페이지 골격, 학생·교사·책 CSV+CRUD, 운영 페이지, 전체 통계, 라벨 PDF
- **Phase 2 (운영 편의)**: 대시보드 요약, 학년 진급/졸업 일괄 처리, 추세 통계
- **Phase 3 (선택)**: ISBN API 자동 채우기, 일괄 반납, CSV 백업, 연체 알림

## 디자인 원칙

- 차분한 톤. 화려함보다 **빠른 작업 처리** 우선 (한 손은 마우스, 한 손은 스캐너).
- 사이드바(5페이지) + 메인 콘텐츠 패턴.
- 한국어 남색 / 영어 초록 컬러 코딩 일관 적용 (KO/EN 배지, 헤더 강조 등).
