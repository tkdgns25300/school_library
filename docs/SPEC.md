# School Library — 기획서

## 개요

- **프로젝트명**: School Library
- **사용처**: 더힘스쿨 수지점 (기독교 대안학교, 1~6학년)
- **목표**: 기존 "르네상스" 도서 대여 시스템을 대체하는 자체 운영 도구
- **사용자**: 교사 다수 (단일 관리자 계정으로 공용 로그인)
- **장서 규모**: 약 300권
- **장비**: USB 바코드 스캐너

## 학교 구조

학년: 1~6.

반(class_section)은 총 3개:

| 반 | 소속 학년 |
|---|---|
| `junior 1` | 1~3학년 학생 일부 |
| `junior 2` | 1~3학년 학생 일부 |
| `senior 1` | 4~6학년 학생 전부 |

학생 데이터는 항상 **학년 + 반** 두 정보를 함께 보유.

## 사용자 모델

- **단일 관리자 계정**: 학교 공용 1개 (Supabase Auth 이메일/비번, 대시보드에서 직접 생성).
- **교사 명단**: 별도 `teachers` 테이블에 이름 등록.
- **담당자 지정**: 대여/반납 시 어느 교사가 처리했는지 선택. 사후 추적/책임 분담 용도.

## 핵심 기능

### 1. 로그인
관리자 계정 이메일/비번 로그인. 비인증 시 모든 페이지에서 `/login` 리다이렉트.

### 2. 교사 관리
- 교사 목록 (이름)
- 추가/수정/삭제 UI

### 3. 학생 관리
- **CSV 일괄 업로드**: `name, grade, class_section, student_number`
- **개별 CRUD UI**
- **반별 조회 (핵심 UX)**: 반 → 학생 목록 → 학생별 대여/연체 현황 즉시 확인
- 검색·필터: 이름 검색, 학년 필터, 반 필터

### 4. 책 관리
- **CSV 일괄 업로드**: `title, author, publisher, grade_level` (바코드는 시스템이 자동 발급)
- **개별 등록**: 제목/저자/출판사/대상학년 → 자동 ID 발급 + 바코드 미리보기
- 수정/삭제 (활성 대여 있는 책은 삭제 차단)
- 검색·필터: 제목/저자, 학년, **미인쇄**, 대여중

### 5. 바코드 발급·미리보기
- 형식: `BK00001`~ (Code128, `bwip-js`)
- 책 등록 시 자동 발급, 등록·상세 화면에서 즉시 프리뷰

### 6. 바코드 라벨 PDF
- 미인쇄 책들을 모아 A4 라벨 시트 PDF 생성
- 인쇄 후 책에 부착 → 시스템에서 `printed=true` 일괄 처리

### 7. 대여
- 학생 선택 (검색 가능 드롭다운, 반별 필터)
- 반납 예정일 (date picker)
- **담당자(교사) 선택**
- 모드: 대여 → 입력 필드 자동 포커스 → **연속 스캔**

### 8. 반납
- 모드: 반납 → **담당자 선택** → 연속 스캔
- 학생 선택 불필요 (스캔된 `book_id`로 활성 loan 자동 매칭)
- 활성 loan 없는 책 스캔 시 경고

### 9. 대여·연체 모니터링 (관리 페이지)

> 통계 리포트가 아니라 **운영용 모니터링**.

- **반별 뷰**: 반 클릭 → 학생 목록 → 학생별 (현재 대여 권수, 연체 권수, 연체일)
- **연체 도서 리스트**: 연체일 정렬, 학생/반/담당자 정보 노출
- **현재 대여 중 도서 리스트**: 검색·필터

## 바코드 전략

- **자체 발급**: 르네상스 바코드 사용 불가 → 시스템이 발급 (Code128, `BK00001`~).
- **출력**: 라벨 PDF → 시판 라벨지 인쇄 → 책에 부착.
- **스캔**: USB 스캐너는 키보드 에뮬레이션 → `<input>` + `onKeyDown`. 별도 드라이버/API 없음.

## 우선순위

### Phase 1 (MVP)
로그인 / 교사·학생·책 관리(CSV+CRUD) / 바코드 미리보기·라벨 PDF / 대여·반납(담당자 지정, 연속 스캔) / 반별 모니터링 + 연체 리스트.

### Phase 2 (운영 편의)
대시보드 요약 / 학년 진급·졸업 일괄 처리 / 책 일괄 작업 / 추세 통계(인기 도서, 학년별 대여 빈도).

### Phase 3 (선택)
ISBN API 자동 채우기 / 일괄 반납 / CSV 백업·내보내기 / 연체 알림.

## 디자인

- 차분한 톤. 화려함보다 **빠른 작업 처리** 우선 (한 손은 마우스, 한 손은 스캐너).
- 사이드바 + 메인 콘텐츠 패턴.
- shadcn/ui 컴포넌트 적극 활용 (Form, Table, Dialog, DatePicker, Combobox).
- 데스크탑 우선, 모바일에서는 조회만 가능하면 충분.

## 비기능 요구

- **반응 속도**: 스캔 후 처리 결과 표시까지 200ms 이내 체감. 연속 스캔이 흐름 끊기지 않을 것.
- **데이터 무결성**: 한 책에 활성 대여는 1개만 (DB 제약). 학생/책 삭제는 활성 대여 있으면 차단.
- **백업**: 학기 단위 CSV 내보내기로 외부 백업 가능 (Phase 3).
